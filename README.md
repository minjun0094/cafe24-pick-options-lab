# Cafe24 골라담기 옵션 UI

Cafe24 베이직 스킨 상품 상세페이지에 골라담기 형태의 옵션 UI를 적용한 과제입니다.
기본 옵션 DOM과 구매 로직은 유지하고, 커스텀 UI에서 원본 옵션 이벤트를 실행하는 방식으로 구현했습니다.

## 적용 URL

- Mobile: https://minjun077.cafe24.com/m/product/detail.html?product_no=11

## 구현 방식

### 1. 옵션 UI

`pick-options-config.js`의 데이터를 바탕으로 개입수 카드와 맛 선택 UI를 생성합니다.

- 10개입, 30개입, 50개입, 100개입 카드
- 가격, 개당 가격, 할인 문구, 추천 뱃지
- 맛별 이미지, 영양정보, 신제품·BEST·품절 상태
- 맛별 수량 선택 및 실시간 검증
- 선택상품 요약, 수량, 가격, 삭제 버튼

선택한 개입수와 맛 수량의 합계가 일치할 때만 선택 완료 버튼이 활성화됩니다.

### 2. 외부 설정 JS

`pick-options-config.js`에서는 다음 값을 관리합니다.

- 적용 상품 번호
- Cafe24 DOM 셀렉터
- UI 문구
- 개입수별 가격과 뱃지
- Cafe24 원본 옵션값
- 맛 UI 데이터

과제에서 지정된 Cafe24 품목 옵션 8개를 기준으로 개입수와 suffix를 동기화했습니다.
맛 선택 데이터는 UI 구성과 선택 결과 표시에 사용하며, 별도의 Cafe24 품목 옵션이나 구매 로직으로 재구현하지 않았습니다.

### 3. Cafe24 옵션 동기화

기본 텍스트 버튼 옵션은 DOM에서 삭제하지 않고, 커스텀 UI 초기화에 성공한 경우에만 화면에서 숨깁니다.

```text
커스텀 옵션 선택 완료
→ 사용 가능한 다음 suffix 확인
→ Cafe24 원본 텍스트 버튼 click()
→ Cafe24가 선택상품 품목 생성
→ Cafe24가 수량·가격·총금액 계산
```

30개입을 처음 선택하면 `30개입_1`, 다시 선택하면 `30개입_2`가 추가됩니다.
두 옵션이 모두 사용되면 추가 선택을 차단합니다.

선택 상태는 별도 카운터만 사용하지 않고 `#totalProducts`의 실제 품목 코드와 수량을 기준으로 다시 계산합니다.
Cafe24가 품목 행을 비동기적으로 추가하거나 삭제하는 경우를 처리하기 위해 `MutationObserver`로 선택상품 목록을 감시합니다.

품목 생성, 가격 계산, 수량 변경, 장바구니 담기와 바로 구매는 Cafe24 기본 DOM과 내장 이벤트가 처리합니다.

## 옵션 설정

```text
10개입_1, 10개입_2
30개입_1, 30개입_2
50개입_1, 50개입_2
100개입_1, 100개입_2
```

| 구성 | 가격 | 개당 가격 |
| --- | ---: | ---: |
| 10개입 | 24,700원 | 2,470원 |
| 30개입 | 70,500원 | 2,350원 |
| 50개입 | 111,500원 | 2,230원 |
| 100개입 | 196,000원 | 1,960원 |

## PC/Mobile 대응

PC와 모바일은 동일한 옵션 데이터와 동기화 방식을 사용합니다.
Cafe24가 생성하는 수량 입력과 구매 버튼 DOM이 달라 플랫폼별 셀렉터와 CSS만 구분했습니다.

모바일에서는 맛 선택 영역이 열려 있는 동안 하단 고정 구매 바를 숨겨 수량 버튼과 구매 버튼의 터치 영역이 겹치지 않도록 했습니다.

## 제한사항

- 맛 선택 정보는 UI 표시용이며 Cafe24 주문 데이터에 별도로 저장하지 않습니다.
- suffix 제한은 상품 상세페이지 옵션 선택 과정에 적용합니다.
- 장바구니 페이지의 수량 변경은 Cafe24 기본 동작을 유지합니다.
- 무료배송 진행 UI는 과제 범위에서 제외하여 비활성화했습니다.
- 상품 이미지, 상세 설명, 배너, 구매 버튼 외형은 수정하지 않았습니다.

1. 운영 요구사항에 맛 정보의 주문 저장이 포함될 경우, Cafe24에서 공식 지원하는 추가 입력 옵션이나 주문 필드를 사용해 연동할 수 있습니다.
