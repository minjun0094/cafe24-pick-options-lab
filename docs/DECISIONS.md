# 설계 결정

## Cafe24 동작을 재구현하지 않는다

커스텀 버튼은 대응하는 Cafe24 옵션 버튼의 `click()`을 호출합니다. 가격 계산, 재고 확인,
품절 처리, 선택 상품 행 추가는 플랫폼 코드가 담당합니다. 플랫폼 동작이 바뀌어도 중복 구현에서
생기는 불일치를 줄일 수 있습니다.

## 설정과 실행 코드를 분리한다

상품 번호, DOM 셀렉터, 옵션값과 문구는 `pick-options-config.js`에 둡니다. 새 상품에 적용할
때 실행 로직을 수정하지 않고 설정만 변경할 수 있습니다. 설정 객체는 깊게 동결해 다른
스크립트가 런타임에 값을 바꾸는 실수를 방지합니다.

## 점진적 향상을 사용한다

기존 옵션 테이블은 커스텀 UI가 정상 생성된 뒤에만 시각적으로 감춥니다. JavaScript 오류,
설정 누락, DOM 변경이 발생하면 기본 Cafe24 옵션이 그대로 남습니다.

## 접근성

개별 선택지는 실제 `button`을 사용합니다. 선택 상태는
`aria-pressed`, 사용 불가능 상태는 `disabled`로 전달합니다. 키보드 초점 표시와
`prefers-reduced-motion`도 지원합니다.

## suffix를 추가 슬롯으로 취급한다

`30개입_1`, `30개입_2`는 서로 다른 구성이 아니라 30개입을 추가할 수 있는 첫 번째와
두 번째 슬롯입니다. 선택상품 목록의 `item_code[]`를 읽어 사용된 슬롯을 판별하고, 카드 클릭
시 아직 사용하지 않은 다음 슬롯을 선택합니다. 행이 삭제되면 해당 슬롯을 다시 사용할 수 있습니다.

## 테스트 상품 DOM 계약

2026-07-11 확인 기준:

- 상품 번호: `11`
- 옵션명: `구성`
- 버튼 목록: `.ec-product-button[product_type="product_option"]`
- 숨겨진 셀렉트: `select#product_option_id1[name="option1"]`
- 옵션: `10/30/50/100개입`, 각 구성 `1/2`
- 옵션값: `P000000L000A`부터 `P000000L000H`
