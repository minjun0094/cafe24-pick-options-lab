# QA 체크리스트

테스트 URL:
`https://minjun077.cafe24.com/product/detail.html?product_no=11`

## 과제 요구사항

- [ ] 1. 상품 상세페이지에 골라담기 형태의 커스텀 옵션 UI가 정상 노출된다.
- [ ] 2. 카드 선택 시 Cafe24 기본 텍스트 버튼과 선택상품 목록이 함께 갱신된다.
- [ ] 3. 반복 클릭·재선택에도 오류나 동일 옵션 중복이 생기지 않는다.
- [ ] 4. `_1`, `_2` 기준으로 개입수별 최대 2회까지만 추가된다.
- [ ] 5. 개입수별 가격·추가금액이 정확하고 8개 품목이 모두 연결된다.
- [ ] 6. 수량 변경, 총금액, 장바구니, 바로 구매가 정상 동작한다.
- [ ] 7. 선택·최대 선택·품절 상태가 사용자에게 명확하게 표시된다.
- [ ] 8. 뱃지·금액·추천 문구·추가 횟수·옵션값이 외부 설정 JS에 분리돼 있다.
- [ ] 9. PC와 모바일 뷰에서 텍스트 및 옵션 행이 깨지지 않는다.

## 예외 및 안전한 실패

- [ ] 선택상품 행을 삭제하면 해당 개입수 카드가 즉시 다시 활성화된다.
- [ ] 품절된 suffix는 추가 가능 횟수에서 제외된다.
- [ ] 대상이 아닌 상품에서는 커스텀 UI가 생성되지 않는다.
- [ ] 초기화 실패 시 Cafe24 기본 옵션 영역이 그대로 노출된다.
- [ ] 콘솔에 오류가 발생하지 않는다.

## 접근성

- [ ] Tab 및 Enter/Space로 모든 활성 카드 선택이 가능하다.
- [ ] 선택 상태가 `aria-pressed`로 노출된다.
- [ ] 최대 횟수 도달 카드가 `disabled`로 노출된다.
- [ ] 키보드 초점 표시가 명확하다.

## 제출용 녹화

- [ ] 30개입 2회 추가 후 3회 차단
- [ ] 선택상품 행 생성 및 삭제 후 카드 재활성화
- [ ] 카드 선택 상태와 Cafe24 기본 구매 흐름 동기화
- [ ] PC에서 모바일 너비로 변경했을 때 반응형 유지
