# QA 체크리스트

테스트 상품:
`https://minjun077.cafe24.com/product/detail.html?product_no=11`

## 기본 동작

- [ ] 페이지 로드 후 수량별 그룹 4개와 각 구성 버튼 2개가 표시된다.
- [ ] 커스텀 UI가 표시되면 기존 Cafe24 옵션 테이블은 화면에서 보이지 않는다.
- [ ] 각 구성 버튼을 누르면 Cafe24의 상품 목록에 동일한 옵션이 추가된다.
- [ ] 추가금액이 있는 30/50/100개입 옵션의 가격이 올바르게 계산된다.
- [ ] 수량 변경 및 삭제 버튼은 기존과 동일하게 동작한다.
- [ ] 장바구니와 바로 구매하기에 선택 옵션이 정확히 전달된다.

## 상태와 예외

- [ ] 품절 옵션은 비활성화되고 `품절` 문구가 표시된다.
- [ ] 이미 선택한 옵션을 다시 선택할 때 Cafe24의 기본 안내가 유지된다.
- [ ] `product_no=11`이 아닌 상품에서는 커스텀 UI가 생성되지 않는다.
- [ ] 설정 스크립트를 제거해도 기존 Cafe24 옵션으로 구매할 수 있다.
- [ ] 옵션 DOM 셀렉터를 의도적으로 틀리게 해도 기존 옵션이 감춰지지 않는다.

## 접근성과 반응형

- [ ] Tab 키만으로 모든 활성 옵션에 접근하고 Enter/Space로 선택할 수 있다.
- [ ] 키보드 초점 테두리가 명확히 보인다.
- [ ] 선택 상태가 `aria-pressed="true"`로 노출된다.
- [ ] 480px 이하에서 옵션 버튼이 한 열로 표시된다.
- [ ] 긴 문구 및 브라우저 200% 확대에서도 내용이 잘리지 않는다.

## 개발자 도구 확인

- [ ] 콘솔 오류가 없다.
- [ ] `window.PickOptions.getOptions()`가 8개 옵션을 반환한다.
- [ ] `pick-options:ready` 이벤트가 한 번만 발생한다.
- [ ] 옵션 선택 시 `pick-options:selection-change` 이벤트가 발생한다.
